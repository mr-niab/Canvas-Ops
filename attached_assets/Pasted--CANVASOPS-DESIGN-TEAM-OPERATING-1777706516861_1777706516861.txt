<!--
=============================================================
CANVASOPS — DESIGN TEAM OPERATING SYSTEM
Replit build brief
=============================================================

PRODUCT SUMMARY
CanvasOps is a web app built specifically for design teams
that include UX/UI designers, user researchers, and service
designers. It is not a generic project management tool — it
is purpose-built around how design teams think, collaborate,
and track work across disciplines.

The core problem it solves: design teams lose track of what
stage a project is at, what resources exist, who the
stakeholders are, and what decisions have been made. Current
tools like Jira, Notion, and DevOps are built for developers
or generalist teams. CanvasOps is built for designers.

CORE PRINCIPLES
1. Projects are the primary unit of work. Every user starts
   from the project list and navigates within a project.
2. Three disciplines connect inside every project:
   UX/UI Design, User Research, and Service Design.
   Each has its own workflow lane but shares one project.
3. Stakeholders are recorded per project with name, role,
   email, and last contact status.
4. A project log captures conversations, decisions, files,
   and emails in a chronological, per-project audit trail.
5. Upcoming sessions replace calendar views — a lightweight
   list of what is coming up, shown below main content.

SCREENS IN THIS PROTOTYPE
- Home: Projects at a glance (primary), My actions today,
  Upcoming sessions (lower section)
- Project detail: Stage tracker, cross-discipline workflow
  lanes, project health, stakeholder summary, log preview
- Stakeholders: Table of project contacts with role, email,
  last contacted, alignment status
- Project log: Chronological log entries with type badges
- Workflow: Kanban-style discipline lanes

DESIGN SYSTEM
- Fonts: Cabinet Grotesk (display/headings), Satoshi (body)
  both via Fontshare CDN — no install needed
- Palette: Warm neutral light theme
  Background #f7f6f2, Surface #f9f8f5, Text #28251d
  Primary teal #01696f, Research purple #7a39bb,
  Service blue #006494
- Single light mode only — no dark mode toggle
- Radius: 14px cards, 12px buttons and tags
- Shadow: warm, low-contrast elevation

TECHNICAL NOTES
- Single HTML file, no framework, no build step
- Navigation uses vanilla JS view switching
- All state is client-side and in-memory
- Fonts loaded from Fontshare CDN (internet required)
- Responsive: sidebar hidden on mobile, grids stack to 1 col
- Expand to multi-page React/Next.js or SvelteKit when ready

NEXT STEPS FOR REPLIT BUILD
1. Convert each view section into a component
2. Add a real data model for projects, tasks, stakeholders,
   log entries using a simple JSON structure or Supabase
3. Add form interactions: create project, add stakeholder,
   add log entry, add task to workflow lane
4. Add filtering on the project list by stage and status
5. Add search across projects, tasks, and stakeholders
=============================================================
-->

<script>
const navBtns = document.querySelectorAll('.nav-btn');

function openView(name) {
  document.querySelectorAll('.view').forEach(v => v.classList.remove('active'));
  const el = document.getElementById('view-' + name);
  if (el) el.classList.add('active');
  navBtns.forEach(b => b.classList.toggle('active', b.dataset.view === name));
  window.scrollTo({ top: 0, behavior: 'smooth' });
}

navBtns.forEach(b => {
  if (b.dataset.view) b.addEventListener('click', () => openView(b.dataset.view));
});

document.querySelectorAll('[data-open-project]').forEach(el => {
  el.addEventListener('click', () => openView('project'));
});
</script>
